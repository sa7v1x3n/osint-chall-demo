03 Stuck? Pull the File's Metadata

If the visual search doesn't land, every photo carries a second trail — its EXIF metadata. This is the forensic fallback.

$ exiftool EXHIBIT-A.PNG Camera Model : [insert camera/phone model] Date/Time Original : [insert capture timestamp] GPS Latitude : [insert latitude] GPS Longitude : [insert longitude] GPS Position : [lat, lng → drop into Google Maps]

⚠️ Many phones/social platforms strip GPS tags on upload — that's why this is the backup method, not the first one.

Entry 01 — What is OSINT?

OSINT means gathering intelligence purely from publicly available sources — search engines, public records, images, metadata — instead of hacking or insider access.

1.1 What does OSINT stand for?

1.2 Name two source types an investigator could use, besides images.

Model Answer: e.g. public records, news archives, forums, government databases, social media bios.

1.3 True or False: OSINT only refers to social media information.

Entry 02 — Reverse Image Search

Search the web using an image instead of text. Usually the first move in any geolocation challenge.

2.1 Name a reverse-image-search tool/engine.

2.2 A result links to a travel blog with the same photo. What do you check next on that page?

Model Answer: The post's text/captions and any other photos on the same page for location names or context clues.

2.3 Why might a reverse image search return nothing, even for a real photo?

Model Answer: The image may be new/never indexed, AI-generated, or too generic to return a unique match.

Entry 03 — Google Lens

Lens identifies landmarks, text, and objects inside an image — useful when a photo is cropped or only shows a partial view.

3.1 What's one advantage Lens has over a normal reverse image search?

Model Answer: It can recognise landmarks/objects within an image, not just match the photo as a whole.

3.2 Lens gives three possible cities for a building. What's your next step?

Model Answer: Cross-reference distinguishing details — signage language, architecture style — against each candidate city.

3.3 List two visual clues inside a photo, besides a landmark, that hint at location.

Model Answer: e.g. language on signage, vehicle/number plate style, vegetation or climate cues.

Entry 04 — EXIF & exiftool

Camera model, timestamp, sometimes GPS — all hidden in the file itself. Your forensic fallback when visual search comes up empty.

4.1 What command shows all metadata in a file named photo.jpg?

4.2 Which two EXIF tags together give an exact location?

4.3 Why do most WhatsApp/Instagram photos have no GPS data, even if the original did?

Model Answer: Most platforms strip EXIF/GPS data on upload to protect user privacy and reduce file size.

4.4 Besides GPS, name another EXIF field that helps guess when a photo was taken.

Entry 05 — Putting It Together

Apply this to Exhibit A directly — no caption, no obvious clues.

5.1 What's your investigation order, step by step, and why that order?

Model Answer: Reverse image search/Lens first (fast, broad) → manual visual clue analysis → EXIF metadata as the slower but sometimes most precise fallback.

5.2 If both visual search and EXIF return nothing, what's a third approach?

Model Answer: Manually analyse visual clues — architecture style, vegetation, road markings, signage language — and compare against known regions.

5.3 What single piece of evidence is strong enough alone vs. evidence you'd double-check?

Model Answer: Strong alone: exact GPS coordinates from EXIF. Worth double-checking: a single Lens visual match, since landmarks can repeat across cities.

Entry 06 — OPSEC Reflection

Geolocating a stranger's photo this easily cuts both ways — it's also a privacy risk to think about for yourself.

6.1 Name one habit to adopt before posting photos online, to reduce your own geolocation risk.

Model Answer: e.g. disable location tagging before posting, or strip EXIF data before sharing.

6.2 Why is platform metadata-stripping a double-edged sword?

Model Answer: It protects everyday users' privacy, but also removes a key forensic trail that investigators rely on — privacy vs. traceability.
